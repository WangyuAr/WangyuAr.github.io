每天一道leetcode34（拓扑排序 BFS/DFS）

發表於 2020-05-17 | 分類於 LeetCode

https://leetcode-cn.com/problems/course-schedule-ii/

现在你总共有 n 门课需要选，记为 0 到 n-1。
在选修某些课程之前需要一些先修课程。 例如，想要学习课程 0 ，你需要先完成课程 1 ，我们用一个匹配来表示他们: [0,1]
给定课程总量以及它们的先决条件，返回你为了学完所有课程所安排的学习顺序。
可能会有多个正确的顺序，你只要返回一种就可以了。如果不可能完成所有课程，返回一个空数组。

输入: 2, [[1,0]] 
输出: [0,1]
解释: 总共有 2 门课程。要学习课程 1，你需要先完成课程 0。因此，正确的课程顺序为 [0,1] 。

输入: 4, [[1,0],[2,0],[3,1],[3,2]]
输出: [0,1,2,3] or [0,2,1,3]
解释: 总共有 4 门课程。要学习课程 3，你应该先完成课程 1 和课程 2。并且课程 1 和课程 2 都应该排在课程 0 之后。
     因此，一个正确的课程顺序是 [0,1,2,3] 。另一个正确的排序是 [0,2,1,3] 。

思路： 拓扑排序问题。先构建邻接表，然后可以通过DFS或者BFS来遍历。
（DFS很麻烦，有很多细节，推荐用BFS进行拓扑排序）。

BFS: 

记录每个结点的入度，将入度为0的结点塞入队列。
每次出队列，该结点相连的结点的入度减1，并判断减去1后入度是否为0， 为0就加入队列。
如果出队列的节点数等于总数说明可以拓扑排序，没有环，否则有环。

BFS代码：

class Solution {
    Map<Integer,List<Integer> > map;
    int numCourses;
    public int[] findOrder(int numCourses, int[][] prerequisites) {
        if (numCourses == 0) return new int[0];
        int [] res = new int[numCourses];
       // 用map构建邻接表
        Map<Integer,List<Integer>> map = new HashMap();

        int[] indegree = new int[numCourses];
        for (int i=0;i<prerequisites.length;i++){
            indegree[prerequisites[i][0]] = indegree[prerequisites[i][0]]+1; // 计算每个节点入度
            if(map.get(prerequisites[i][1])!=null){
                List<Integer> l = map.get(prerequisites[i][1]);
                l.add(prerequisites[i][0]);
                map.put(prerequisites[i][1], l);
            }else{
                List<Integer> l = new ArrayList();
                l.add(prerequisites[i][0]);
                map.put(prerequisites[i][1],l);
            }
        }

        Queue<Integer> queue = new LinkedList<>();
        //找入度为0的结点
        for (int i = 0; i < indegree.length; i++) {
            if (indegree[i] == 0) queue.offer(i);
        }
        int cnt = 0;
        while(!queue.isEmpty()){
            int curr = queue.poll();
            res[cnt++] = curr;
            List<Integer> l = map.get(curr);
            if(l!=null){
                for (int xx : l){
                    // 入度-1，并判断是否为0 
                    indegree[xx] = indegree[xx] -1;
                    if(indegree[xx] == 0){
                        queue.offer(xx);
                    }
                }
            }
        }
         if (cnt == numCourses) return res;
         return new int[0];
    }
}

给定一个整数数组和一个整数 k，你需要找到该数组中和为 k 的连续的子数组的个数。
https://leetcode-cn.com/problems/subarray-sum-equals-k/

输入:nums = [1,1,1], k = 2
输出: 2 , [1,1] 与 [1,1] 为两种不同的情况。

前缀和的思路有点类似动态规划。

用一个数组presum记录，presum[i]表示nums[0]…nums[i]的和。

 // 计算前缀和数组
int[] preSum = new int[len + 1];
preSum[0] = 0;
for (int i = 0; i < len; i++) {
    preSum[i + 1] = preSum[i] + nums[i];
}

和为K的连续数字，即preSum[left] - preSum[right] == k

for (int left = 0; left < len; left++) {
    for (int right = left; right < len; right++) {
        // 区间和 [left..right]
        if (preSum[right + 1] - preSum[left] == k) {
            count++; // 满足条件的数目
        }
    }
}

为了解决两层for循环，使用hashmap进行优化：key值为当前前缀和，value是key值的次数

public int subarraySum(int[] nums, int k) {
        Map<Integer,Integer> map = new HashMap();
        int pre = 0; int res =0;
        map.put(0,1);
        for(int i =0;i<nums.length;i++){
            pre = pre + nums[i];
            if(map.containsKey(pre - k)){ // 存在pre - k，就加上出现的次数
                res = res + map.get(pre-k);
            }
            map.put(pre, map.getOrDefault(pre,0)+1);//pre出现次数+1
        }
        return res;
    }

每天一道leetcode124 (递归回溯最长路径)

發表於 2020-05-14 | 分類於 LeetCode

今天每日一题很简单，所以另外做了一道，Leecode124题。

给定一个非空二叉树，返回其最大路径和。
本题中，路径被定义为一条从树中任意节点出发，达到任意节点的序列。该路径至少包含一个节点，且不一定经过根节点。

输入: [1,2,3]

       1
      / \
     2   3

输出: 6

输入: [-10,9,20,null,null,15,7]

   -10
   / \
  9  20
    /  \
   15   7

输出: 42

思路：从根节点出发 深度优先遍历左右子树，得到左右子树的最大路径 left、right。

那么 max(root) = root.val + max(left,right);

如果必须经过根节点，那么结果就是max(root)。

现在可以不经过，这意味着我们要修改上面的函数，在每一步都检查哪种选择更好：是继续当前路径或者以当前节点作为最高节点计算新的路径。

  1. 初始化 max_sum 为最小可能的整数并调用函数 max_gain(node = root)。

  2. 实现 max_gain(node) 检查是继续旧路径还是开始新路径：

    • 边界情况：如果节点为空，那么最大权值是 0 。

    • 对该节点的所有孩子递归调用 max，计算从左右子树的最大权值：

      `left     = max     (max(node.left), 0)和right = max(max(node.right),0)。`

    • 检查是维护旧路径还是创建新路径。创建新路径的权值是：
      price_newpath = node.val + left + right，当新路径更好的时候更新 全局变量max_sum。

    • 对于递归返回的到当前节点的一条最大路径，计算结果为：
      node.val + max(left_gain, right_gain)

代码： 

class Solution {
    int ans = Integer.MIN_VALUE;
    public int maxPathSum(TreeNode root) {
        dfs(root);
        return ans;
    }
    public int dfs(TreeNode root){
        if(root==null)
            return 0;
        int left = Math.max(dfs(root.left),0);
        int right =  Math.max(dfs(root.right),0); 
        int curpath = root.val + Math.max(left,right);
        int newpath = root.val + left + right;
        ans = Math.max(ans,Math.max(curpath ,newpath));
        return curpath;
    }
}

Leetcode 129题 求根到叶子节点数字之和

https://leetcode-cn.com/problems/sum-root-to-leaf-numbers

输入: [1,2,3]
    1
   / \
  2   3
输出: 25
解释:
从根到叶子节点路径 1->2 代表数字 12.
从根到叶子节点路径 1->3 代表数字 13.
因此，数字总和 = 12 + 13 = 25.


输入: [4,9,0,5,1]
    4
   / \
  9   0
 / \
5   1
输出: 1026
解释:
从根到叶子节点路径 4->9->5 代表数字 495.
从根到叶子节点路径 4->9->1 代表数字 491.
从根到叶子节点路径 4->0 代表数字 40.
因此，数字总和 = 495 + 491 + 40 = 1026.

当前结点若是叶子结点，就把这条路径上的值加到全局变量ans上

class Solution {
    int ans = 0;
    public int sumNumbers(TreeNode root) {
        dfs(root, 0);
        return ans;
    }
    public int dfs(TreeNode root, int temp){
        if(root == null)
            return temp;
        if(root.left == null && root.right == null){
            ans += temp *10 + root.val;
            return temp;
        }

        temp = temp *10 +root.val;

        dfs(root.left, temp);

        dfs(root.right, temp);

        return temp;
    }
}

今天的每日一题很简单，就是树的层次遍历。

给你一个二叉树，请你返回其按 层序遍历 得到的节点值。 （即逐层地，从左到右访问所有节点）。

示例：
二叉树：[3,9,20,null,null,15,7],

    3
   / \
  9  20
    /  \
   15   7
返回其层次遍历结果：

[
  [3],
  [9,20],
  [15,7]
]

思路： 利用队列，先进先出，注意记录当前层和当前层的节点数。

代码：

class Solution {
    public List<List<Integer>> levelOrder(TreeNode root) {
        List<List<Integer>> res = new ArrayList();

        if(root==null)
            return res;

        Queue<TreeNode> q = new LinkedList();
        q.add(root);
        int curlevel = 0;
        while(!q.isEmpty()){
            int level_len = q.size();//每一层的结点个数
            res.add(new ArrayList());
            for(int i =0;i<level_len;i++){
                TreeNode t = q.poll();
                res.get(curlevel).add(t.val);
                if(t.left!=null) // 每poll一个就add他的子节点进来
                    q.add(t.left);
                if(t.right!=null)
                    q.add(t.right);
            }
            curlevel++;

        }
        return res;
    }
}

今天的leetcode每日一题是第236题：二叉树的最近公共祖先

给定一个二叉树, 找到该树中两个指定节点的最近公共祖先。
https://leetcode-cn.com/problems/lowest-common-ancestor-of-a-binary-tree/

输入: root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 1
输出: 3
解释: 节点 5 和节点 1 的最近公共祖先是节点 3。

思路： 这题显然就是递归回溯，过程是

  1. 递归出口：当前结点是叶子节点
  2. 依次递归root的左子树 右子树，如果p、q分别在左右子树上那就是root,如果root是p或者q，并且另一个目标数字在root的左或者右子树上，也返回root

代码：（实质上可以看作是一个后序遍历）

class Solution {
    TreeNode p;
    TreeNode q;

    TreeNode res;

    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        this.p = p;
        this.q = q;
        dfs(root);
        return res;
    }
    public boolean dfs(TreeNode root){
        if(root== null){ // 递归出口，是根节点
            return false;
        }
        boolean left = dfs(root.left);
        boolean right = dfs(root.right);
        if(left&&right || (root.val == p.val || root.val == q.val)&&(left||right))
            res = root;

        return (left||right || (root.val == p.val || root.val == q.val));
    }
}

还有一道递归回溯，leetcode周赛的一道题目。https://leetcode-cn.com/problems/minimum-time-to-collect-all-apples-in-a-tree/

思路： 也是后序遍历。

代码：

public class Leetcode5046 {

    public int minTime(int n, int[][] edges, List<Boolean> hasApple) {
        Map<Integer, List<Integer>> map = new HashMap();
        for (int i = 0; i < edges.length; i++) {
            int[] t = edges[i];
            List<Integer> l = null;
            if (map.get(t[0]) == null) {
                l = new ArrayList();
                l.add(t[1]);
            } else {
                l = map.get(t[0]);
                l.add(t[1]);
            }
            map.put(t[0], l);
        }
        int res = 0;
        List<Integer> children = map.get(0);
        for (int childIndex : children){
            res += dfs(n, map, hasApple, childIndex);
        }
        return res;
    }

    public int dfs(int n, Map<Integer, List<Integer>> map, List<Boolean> hasApple, int curNode) {
        List<Integer> l = map.get(curNode);
        if (l == null || l.size() == 0) {
            if (hasApple.get(curNode) == true) {
                return 2;
            }
            return 0;
        }
        int ans = 0;//注意！！！！！！！！！！！

        for (int i = 0; i < l.size(); i++) {
            ans = ans + dfs(n, map, hasApple, l.get(i));
        }

        if (ans > 0 ) {
            return ans + 2;
        } else if (hasApple.get(curNode) == true) {
            return 2;
        }else{
            return 0;
        }
    }
}

给定不同面额的硬币 coins 和一个总金额 amount。编写一个函数来计算可以凑成总金额所需的最少的硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。

输入: coins = [1, 2, 5], amount = 11
输出: 3 
解释: 11 = 5 + 5 + 1

输入: coins = [2], amount = 3
输出: -1

思路： 首先想到的肯定是递归回溯深搜啊，可惜超时了。超时代码：

class Solution {
    int[] coins;
    int amount;
    int res = 10000;
    public int coinChange(int[] coins, int amount) {
        if(amount==0)
            return 0;
        this.coins = coins;
        this.amount = amount;
        for(int i =0;i<coins.length;i++){
            if(coins[i]>amount)
                continue;
            dfs(coins[i],0);
        }
        if(res==10000)
            return -1;
        else
            return res+1;
    }
    public void dfs(int curAmount, int curNum) {
        if(curAmount>amount){
            return;
        }
        if(curAmount == amount){
            if(curNum<res)
                res = curNum;
            return ;
        }
        for(int i =0;i<coins.length;i++){
            if(coins[i]>amount)
                continue;
            dfs(curAmount+coins[i], curNum+1);
        }
    }
}

给定一个按照升序排列的整数数组 nums，和一个目标值 target。找出给定目标值在数组中的开始位置和结束位置。

你的算法时间复杂度必须是 O(log n) 级别。

如果数组中不存在目标值，返回 [-1, -1]。

输入: nums = [5,7,7,8,8,10], target = 8
输出: [3,4]

输入: nums = [5,7,7,8,8,10], target = 6
输出: [-1,-1]

#思路
显然，这道题第一反应是二分搜索，当时没想起来怎么做，就写的二分找到之后在往前后线性查找，在极端情况(nums中所有数字都是target的时候)下复杂度O（N）。

以查找右侧端点为例,找到nums[mid] == target之后不返回，继续放缩边界：

int right_bound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid - 1;
        } else if (nums[mid] == target) {
            // 这里改成收缩左侧边界即可
            left = mid + 1;
        }
    }
    // 这里改为检查 right 越界的情况，见下图
    if (right < 0 || nums[right] != target)
        return -1;
    return right;
}

注意：
right = nums.length - 1;对应left <= right而不是left < right

链接：https://leetcode-cn.com/problems/find-first-and-last-position-of-element-in-sorted-array/solution/er-fen-cha-zhao-suan-fa-xi-jie-xiang-jie-by-labula/

二分算法的总结：(注意查找区间的开闭！！！)

第一个，最基本的二分查找算法：

因为我们初始化 right = nums.length - 1
所以决定了我们的「搜索区间」是 [left, right]
所以决定了 while (left <= right)
同时也决定了 left = mid+1 和 right = mid-1

因为我们只需找到一个 target 的索引即可
所以当 nums[mid] == target 时可以立即返回

第二个，寻找左侧边界的二分查找：

因为我们初始化 right = nums.length - 1
所以决定了我们的「搜索区间」是 [left, right]
所以决定了 while (left <= right)
同时也决定了 left = mid + 1 和 right = mid - 1

因为我们需找到 target 的最左侧索引
所以当 nums[mid] == target 时不要立即返回
而要收紧右侧边界以锁定左侧边界       right = mid - 1;

#SpringMVC
DispatcherServlet会根据一个XML文件来加载Spring应用上下文，这个XML文件名是servlet名-servlet.xml。对于其他一些Bean，是否也要配置在该xml文件中？

我们将Spring配置分为了多个xml文件：一个用于服务层、一个用于持久层还有一个用于数据源配置。通过ContextLoaderListener来加载其他的配置文件，ContextLoaderListener是一个Servlet监听器，为了使用它需要在web.xml中添加声明：

<listener>
    <listener-class>
        org.springframework.web.context.ContextLoaderListener
    </listener-class>
</listener>

我们必须要告诉ContextLoaderListener需要加载哪些配置文件。如果没有指定，上下文加载器会查找/WEB-INF/applicationContext.xml这个Spring配置文件。
为了指定ContextLoaderListener需要加载哪些配置文件，需要在servlet上下文中配置contextConfigLocation参数：

<context-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>
        //配置文件路径
    </param-value>
</context-param>
